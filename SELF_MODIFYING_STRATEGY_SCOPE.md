# Self-Modifying Code/Strategy System - Technical Scope

## Overview
Design a system that allows the AI strategy learner to modify its own trading strategies and parameters based on learning outcomes, without requiring manual PR merges.

## Current Implementation Status ✅
- **Strategy Mode System**: Implemented 5 learning strategies (chaos_mode, micro_gains, momentum_chase, contrarian, technical_pure)
- **Hourly Learning**: Active strategy switching every 1-4 hours based on performance
- **Parameter Auto-Apply**: Safe parameter adjustments within guardrails
- **Performance Tracking**: Timeline and insights collection for data-driven decisions

## Proposed Self-Modifying Architecture

### 1. Strategy Override System (RECOMMENDED - LOW RISK)
**File**: `strategy_overrides.json`
```json
{
  "version": 1,
  "timestamp": "2026-03-04T22:00:00Z",
  "overrides": {
    "sentiment_buy_threshold": 6.5,
    "take_profit_pct": 15.0,
    "stop_loss_pct": -8.0,
    "strategy_mode": "momentum_chase",
    "custom_rules": {
      "rsi_override": {"enabled": true, "threshold": 65},
      "macd_weight": 1.5
    }
  },
  "reason": "Detected high volatility period, adjusting for momentum trading",
  "performance_target": {"win_rate": 65, "timeline": "24h"}
}
```

**Implementation**:
- Learner writes to `strategy_overrides.json` when significant learning occurs
- Trading engine reads overrides on each cycle 
- Web interface shows active overrides with revert options
- Git-tracked but safe (JSON changes, not code changes)

### 2. Dynamic Rule Engine (MEDIUM RISK)
**File**: `dynamic_rules.py` (generated)
```python
# Auto-generated by StrategyLearner at 2026-03-04T22:00:00Z
# Reason: Learned RSI patterns for better entries

def get_custom_buy_multiplier(symbol, score, technical):
    """Custom logic learned from recent trade patterns."""
    multiplier = 1.0
    
    # Learned rule: Lower RSI + high sentiment = better outcomes
    if technical.get('rsi', 50) < 35 and score > 7.0:
        multiplier *= 1.3
    
    # Learned rule: MACD bullish + momentum = good for BTC
    if symbol == 'BTC' and technical.get('macd', {}).get('bullish'):
        multiplier *= 1.2
        
    return multiplier
```

**Pros**: More flexible than JSON, can implement complex learned logic
**Cons**: Code generation risk, requires careful sandboxing

### 3. Plugin System (HIGH FLEXIBILITY)
**Directory**: `strategies/learned/`
- Each learning session generates a new strategy plugin
- Hot-loaded by the trading engine
- Versioned and rollback-capable
- Can be enabled/disabled independently

### 4. Neural Parameter Evolution (ADVANCED)
- Machine learning model that evolves parameters
- Uses genetic algorithms or reinforcement learning
- Updates model weights based on trade outcomes
- Requires more sophisticated infrastructure

## Security & Safety Measures

### 1. Sandbox Execution
- All generated code runs in restricted environment
- No file system access beyond designated directories
- No network access from generated code
- Time and memory limits on execution

### 2. Guardrails
- Parameter bounds checking (already implemented)
- Maximum change limits per time period
- Rollback mechanism for poor performance
- Human override capability

### 3. Validation
- Generated code syntax checking
- Static analysis for dangerous patterns
- Performance simulation before live application
- Gradual rollout (shadow testing first)

## Conclusion

**Current implementation (Phase 1) is sufficient** for aggressive learning and experimentation. The strategy mode system with hourly switching and parameter auto-adjustment provides significant learning capabilities without code modification risks.

**Recommended next step**: Implement Phase 2 (Enhanced JSON Overrides) only if current system proves insufficient after extended operation.

The key insight is that **parameter and threshold adjustments within guardrails** can provide 80% of the learning benefits with 20% of the risk compared to full code modification.
